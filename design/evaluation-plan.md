# Stock Agent Evaluation Plan

## 评测目标

Phase 6 的评测要回答：**这个 Agent 的设计是否正确？哪些地方需要改进？**

不是评"预测准不准"，而是评"Agent 的行为是否符合设计意图"。

## 评测维度

| 维度 | 评什么 | 权重 |
|------|--------|------|
| 工具选择 | Agent 是否选择了正确的工具组合 | 高 |
| 参数提取 | Agent 是否从用户输入中正确提取了工具参数 | 高 |
| 数据呈现 | 输出是否准确、结构清晰 | 中 |
| 错误处理 | 工具失败时 Agent 的回复是否合理 | 中 |
| 边界拒答 | 超出能力范围时是否正确拒答 | 高 |
| 多轮连贯 | 是否能记住上下文，正确关联历史 | 中 |

## 评分标准

每条用例按 3 级评分：

| 等级 | 标准 |
|------|------|
| ✅ Pass | 完全符合期望 |
| ⚠️ Partial | 核心意图达到，但细节有偏差 |
| ❌ Fail | 未达到期望，或出现错误行为 |

## 评测流程

```
1. 准备评测用例（14 条）
2. 逐条执行，记录 Agent 的完整输出
3. 对每条用例评分（Pass / Partial / Fail）
4. 统计各维度的通过率
5. 分析失败案例，归类根因：
   - 工具选择错误 → 提示词问题
   - 参数提取错误 → 用户输入理解问题
   - 输出格式问题 → 提示词格式要求不明确
   - 边界问题 → 提示词边界定义不清晰
6. 根据分析结果迭代提示词或工具定义
```

## 与 Phase 6 的关系

Phase 6 的评测方法论：
- 7 个判断力问题（能否识别 Agent 的设计缺陷）
- 失败案例归类和根因分析
- 迭代优化后再评测，对比前后数据

本评测计划的 14 条用例就是 Phase 6 评测的输入。
